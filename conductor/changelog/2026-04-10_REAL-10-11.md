# REAL-10 & REAL-11 · Expose Analytics to MCP

**Timestamp**: 23:15:00 Local Time  
**Author**: Ralph  

## Why
To enable AI agents connecting via Model Context Protocol (MCP) to perform higher-level project management tasks, the server now exposes portfolio health and team workload analytics. This allows agents to identify risks and bottlenecks autonomously.

## What Changed
- `backend/app/crud/crud_user.py`: Added `remove` method to support administrative user deletion via MCP.
- `backend/app/api/api_v1/endpoints/mcp.py`:
    - Implemented `get_portfolio_health` tool: returns a formatted report of project health scores, overdue counts, and risks.
    - Implemented `get_team_workload` tool: returns a report of daily effort per user over 14 days, flagging overallocation.
    - Implemented `delete_user`, `delete_project`, `create_subtask`, and `assign_project_member` tools to fill gaps in the MCP interface.
- `backend/test_mcp_standalone.py`: Updated and expanded the test suite to verify all new tools.
