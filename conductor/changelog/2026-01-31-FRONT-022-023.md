# Changelog: Subtask Metadata Parity (FRONT-022, FRONT-023)

- **Backend**: Updated `SubtaskShortCreate` schema to include `assignee_ids`.
- **Frontend (`SubtaskManager`)**:

    - Improved UX with a toggleable "Add Subtask" form.
- **Frontend (`TaskForm`)**:
    - Completely redesigned the initial subtasks list into a grid/card layout.
    - Added Start Date, Due Date, Priority, and Assignee selection for each initial subtask.
- **PRD**: Updated Section 9.4 to mandate metadata parity during subtask creation.
