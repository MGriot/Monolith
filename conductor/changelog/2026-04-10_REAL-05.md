# REAL-05 · Implement Frontend WebSocket client

**Timestamp**: 15:35:00 Local Time  
**Author**: Ralph  

## Why
Transitioned the notification system from a polling-based approach to a real-time event-driven approach. This reduces server load and provides an instantaneous user experience.

## What Changed
- `frontend/src/components/notification-popover.tsx`:
    - Removed `refetchInterval` from the notifications query.
    - Implemented a `useEffect` hook to manage a WebSocket connection to `/api/v1/notifications/ws/{token}`.
    - Added logic to show a toast notification and invalidate the cache when a `new_notification` message is received.
    - Included a heartbeat mechanism (ping/pong) to maintain the connection.
