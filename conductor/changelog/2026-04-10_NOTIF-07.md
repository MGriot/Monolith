# [NOTIF-07] Integrate weekly summary scheduler

**Timestamp**: 13:55:00 Local Time  
**Author**: Ralph  

## Why
Automated weekly summaries need a reliable trigger mechanism to ensure they are dispatched every Monday without manual intervention.

## What Changed
- `backend/app/core/scheduler.py`: Implemented a background loop that checks the current time every hour and triggers `dispatch_all_summaries` if it's Monday 08:00 UTC and it hasn't run yet this week.
- `backend/app/main.py`: Integrated `start_scheduler()` in the FastAPI startup event.
- Added `last_summary_run.txt` persistence to prevent multiple runs within the same hour/week window.

## Verification
- Verified `start_scheduler` is called in `backend/app/main.py`.
- Verified the loop logic in `backend/app/core/scheduler.py` via code review.
- Checked logs for "Weekly summary scheduler started."

## Notes
- Persistence is currently file-based; for high-availability clusters, a database-backed or Redis-backed lock should be used.
