# Changelog - FRONT-072 & BACK-038

## Fix: Task Conclusion Date Persistence & Display

Fixed multiple issues related to task conclusion dates (completed_at) across the frontend and backend.

### Changes:

#### Frontend:
- **`project-gantt.tsx`**:
    - Renamed internal `conclusionDate` variable to use `completed_at` to match the data model.
    - Removed the "today" fallback for completed tasks without a set date, which was causing misleading visualization.
- **`task-form.tsx`**:
    - Implemented date formatting logic in `useForm` default values to convert ISO timestamps to `YYYY-MM-DD`. This ensures dates are correctly displayed in HTML date inputs when editing tasks.
- **`task-create-dialog.tsx`**:
    - Ensured all date fields are converted to ISO strings before submission during task creation.

#### Backend:
- **`crud_task.py`**:
    - Refined the `update` method to automatically set `completed_at` to the current UTC time when a task's status is changed to `DONE` (if no date is provided in the request).
    - Added logic to clear `completed_at` when a task's status is moved away from `DONE`.

### Verification:
- Ran `npx tsc --noEmit` in the `frontend` directory; all types pass.
- Verified that `completed_at` is now correctly displayed in the `TaskForm` when editing.
- Verified that the Gantt chart correctly shows completion markers only when a completion date is present.
