**Timestamp**: 16:45:00 Local Time

## Why
Users need a fast and centralized way to navigate the application. A global search feature allows users to quickly find Projects, Tasks, and Ideas from any page.

## What Changed
- `backend/app/schemas/search.py`: Created `SearchResult` schema for unified results.
- `backend/app/api/api_v1/endpoints/search.py`: Implemented a new search endpoint that queries Projects, Tasks, and Ideas in parallel.
- `backend/app/api/api_v1/api.py`: Registered the `/search` router.
- `backend/app/api/api_v1/endpoints/mcp.py`: Added `search_projects` and `unified_search` tools for AI agent access.
- `frontend/src/components/layout.tsx`:
    - Integrated a global search input in the header.
    - Added a real-time results dropdown with category icons and status badges.
    - Implemented `Ctrl+K` keyboard shortcut to focus the search.
    - Added click-outside and navigation logic to manage the search state.

## Notes
- The search endpoint uses `ilike` for simple string matching. Performance can be further improved with PostgreSQL Full Text Search or an external indexer (e.g., Meilisearch) if the database grows significantly.
- The UI uses category-specific colors (Blue for Projects, Slate for Tasks, Amber for Ideas) to help users distinguish result types at a glance.
