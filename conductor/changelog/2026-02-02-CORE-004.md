# Changelog: CORE-004 - CRUD: Advanced Dependency Management

## Changes
- Created `backend/app/crud/crud_dependency.py`:
    - Implemented `CRUDDependency` with built-in circular dependency validation using `has_cycle`.
- Updated `backend/app/crud/crud_task.py`:
    - Refactored `CRUDTask.get` and `CRUDTask.get_multi_by_project` to use `selectinload` for the new `blocked_by` and `blocking` relationships.
    - Updated `is_blocked_by_recursive` and `check_for_active_blockers` to use the `Dependency` table as the primary source of truth, while maintaining fallback for legacy `blocked_by_ids`.
- Updated `backend/app/api/api_v1/endpoints/tasks.py`:
    - Added `POST /tasks/{task_id}/dependencies` to create new typed dependencies with lag.
    - Added `DELETE /tasks/{task_id}/dependencies/{predecessor_id}` to remove specific dependencies.

## Verification
- Created and executed `backend/tests/test_crud_dependencies_standalone.py` inside the container:
    - Confirmed creation of typed dependencies (FS, SS, etc.).
    - Confirmed that circular dependencies (A->B, B->A) are rejected with a `ValueError`.
    - Confirmed that `Task` retrieval correctly includes related `Dependency` objects via relationships.
