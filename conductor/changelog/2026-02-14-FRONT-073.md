# Changelog - FRONT-073

## Fix: Task Edit Form completed_at initialization

Fixed an issue where the "Conclusion Date" was not appearing in the task edit form even if it was present in the database.

### Changes:
- **`project-detail.tsx`**: Added the missing `completed_at` field to the `initialValues` object passed to `TaskForm`. This ensures that when a user clicks to edit a task, the existing completion date is loaded from the task object into the form state.

### Verification:
- Verified that the `completed_at` field is now correctly passed to the form.
- The `TaskForm` component (updated in `FRONT-072`) now receives this value and formats it correctly for the HTML date input.
