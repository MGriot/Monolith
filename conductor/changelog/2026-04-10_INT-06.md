# INT-06 · Resource Workload Leveling Logic

**Timestamp**: 20:15:00 Local Time  
**Author**: Ralph  

## Why
Project managers need to identify team members who are overloaded with tasks. By calculating daily effort based on task durations, the system can proactively flag overallocation (e.g., more than 8 hours of work assigned per day).

## What Changed
- `backend/app/schemas/workload.py`: Created schemas for the workload response, including per-user and per-day effort breakdown.
- `backend/app/api/api_v1/endpoints/dashboard.py`: Implemented the `/team-workload` endpoint. It aggregates all active tasks with start/due dates, calculates daily effort (assuming 8h total per task distributed evenly), and identifies overallocated users.
- `backend/test_workload_logic.py`: Verified the calculation logic with a unit test.

## Notes
- The logic assumes an 8-hour workday.
- It evenly distributes the task effort across its duration.
- Only tasks with both `start_date` and `due_date` that are not `Done` are included in the calculation.
