# INT-05 · Webhook Configuration API & UI

**Timestamp**: 19:45:00 Local Time  
**Author**: Ralph  

## Why
To make the webhook service usable, users need a way to store and manage their Slack/Discord URLs. This task adds the database storage and a settings UI for personal webhook configuration.

## What Changed
- `backend/app/models/webhook.py`: Created `Webhook` model with fields for name, url, provider, and active status. Linked to User (owner) and optionally Project.
- `backend/app/schemas/webhook.py`: Defined Pydantic schemas for webhook operations.
- `backend/app/crud/crud_webhook.py`: Implemented CRUD logic for webhooks.
- `backend/app/api/api_v1/endpoints/webhooks.py`: Created API endpoints for managing webhooks and a `/test` endpoint to verify delivery.
- `backend/app/api/api_v1/api.py`: Registered the webhooks router.
- `frontend/src/pages/settings.tsx`: Added a "Webhooks" section where users can add, delete, and test their integrations.
