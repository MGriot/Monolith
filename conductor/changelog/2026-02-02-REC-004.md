# Changelog: REC-004 - Logic: Recursive Status & WBS

## Changes
- Refactored `backend/app/core/wbs.py`:
    - Made `apply_wbs_codes` recursive to support N-level task trees.
- Refactored `backend/app/crud/crud_task.py`:
    - Implemented `update_parent_status_recursive` to bubble status changes (TODO -> IN_PROGRESS -> DONE) up to the root task.
    - Updated `create` to recursively create nested subtasks from request bodies.
    - Added `parent_id` filtering to `get_multi_by_project`.
    - Deprecated separate `CRUDSubtask` logic, pointing it to the unified `task` CRUD.
- Updated `backend/app/api/api_v1/endpoints/tasks.py`:
    - Added `parent_id` query parameter to `GET /tasks/`.
- Updated `backend/app/api/api_v1/endpoints/subtasks.py`:
    - Redirected all operations to use the unified recursive `CRUDTask`.

## Verification
- Created `backend/tests/test_recursive_logic.py`:
    - Verified that creating a 3-level task structure works.
    - Verified that updating a leaf task status (Level 2) correctly updates Level 1 and Level 0 (Root) statuses.
- All tests passed.
